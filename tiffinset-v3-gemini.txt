2. SYSTEM ARCHITECTURE
======================================================================

2.1 Data flow
-------------

User sends voice note/text on Telegram → webhook POST to api.tiffinset.com/webhook/telegram → server returns 200 immediately → if voice: download .ogg from Telegram, send to Whisper, get text → look up sender in user_profiles → load conversation history from Redis → build role-specific system prompt → send to Gemini 2.5 Flash with function declarations → Gemini decides function calls (recipe search, YouTube, mock Swiggy) → execute functions, send results back as functionResponse → Gemini generates final text → route response to kitchen members by role → send via Telegram → log event, update state.

2.2 Transport abstraction
-------------------------

All messaging platform code lives behind a transport interface. Core pipeline imports from src/transport/index.js. Currently points to src/transport/telegram.js. When WhatsApp is approved, add src/transport/whatsapp.js and change one config line.

2.3 AI layer: Gemini 2.5 Flash
------------------------------

Gemini is the AI orchestrator. It uses function calling (tool use) to decide what actions to take. Key differences from Claude that affect the code:

| Aspect                   | Gemini format                                                |
|--------------------------|--------------------------------------------------------------|
| API endpoint             | POST generativelanguage.googleapis.com/v1beta/models/gemini- |
| Auth                     | API key as query parameter: ?key={GEMINI_API_KEY}            |
| System prompt            | systemInstruction field in request body                      |
| Conversation history     | contents[] array with role 'user' and 'model'                |
| Tool definitions         | tools[].functionDeclarations[] with name, description, param |
| Tool call in response    | candidate.content.parts[].functionCall.name and .args        |
| Sending tool result back | role: 'user', parts[].functionResponse.name and .response    |
| End of turn              | candidate.content.parts[].text (no functionCall = done)      |
| Finish reason            | candidate.finishReason: 'STOP' (text) or absent (tool call p |

5. GEMINI API — FUNCTION CALLING REFERENCE
======================================================================

This section is the implementation reference for the AI orchestrator. Every code prompt in the build playbook references this.

5.1 Basic call structure
------------------------

```
const GEMINI_URL = 'https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent';
async function callGemini(systemPrompt, contents, tools) {
  const body = {
    systemInstruction: { parts: [{ text: systemPrompt }] },
    contents: contents,  // [{role:'user',parts:[{text:'...'}]}, {role:'model',parts:[{text:'...'}]}]
    tools: tools ? [{ functionDeclarations: tools }] : undefined,
  };
  const res = await fetch(`${GEMINI_URL}?key=${config.GEMINI_API_KEY}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(body),
  });
  const data = await res.json();
  const candidate = data.candidates[0];
  const parts = candidate.content.parts;
  // Check if Gemini wants to call a function
  const fnCall = parts.find(p => p.functionCall);
  if (fnCall) {
    return {
      type: 'function_call',
      name: fnCall.functionCall.name,
      args: fnCall.functionCall.args,
      rawParts: parts  // may contain text + functionCall
    };
  }
  // Text response
  const text = parts.filter(p => p.text).map(p => p.text).join('');
  return { type: 'text', text };
}
```

5.3 Sending function results back
---------------------------------

```
// After executing a function, send result back to Gemini:
contents.push({
  role: 'model',
  parts: [{ functionCall: { name: 'search_recipe', args: { dish_name: 'paneer tikka' } } }]
});
contents.push({
  role: 'user',
  parts: [{
    functionResponse: {
      name: 'search_recipe',
      response: { recipe: { ingredients: [...], steps: [...] } }
    }
  }]
});
// Then call Gemini again with updated contents
```

5.4 Agentic loop
----------------

```
async function processMessage(chatId, text) {
  const user = await getUser(chatId);
  const kitchen = await getKitchen(user.kitchen_id);
  const systemPrompt = buildSystemPrompt(user, kitchen);
  const history = await loadHistory(chatId); // from Redis
  history.push({ role: 'user', parts: [{ text }] });
  let iterations = 0;
  while (iterations < 5) {
    const result = await callGemini(systemPrompt, history, toolDefinitions);
    if (result.type === 'text') {
      history.push({ role: 'model', parts: [{ text: result.text }] });
      await saveHistory(chatId, history);
      return result.text;
    }
    // Function call: execute and feed result back
    history.push({ role: 'model', parts: result.rawParts });
    const fnResult = await executeTool(result.name, result.args, { chatId, kitchen });
    history.push({
      role: 'user',
      parts: [{ functionResponse: { name: result.name, response: fnResult } }]
    });
    iterations++;
  }
  await saveHistory(chatId, history);
  return 'Processing mein thoda time lag raha hai. Thodi der mein try karo.';
}
```

7. CORE PIPELINE + TOOLS
======================================================================

| Stage            | Action                          | On failure                 |
|------------------|---------------------------------|----------------------------|
| 1. Receive       | Webhook POST from Telegram      | Return 200, skip malformed |
| 2. Verify        | Check secret_token header       | Reject 403                 |
| 3. Deduplicate   | Check msgid:{id} in Redis       | Skip if seen               |
| 4. Rate limit    | Check rate:{chatId}             | Throttle if >20/min        |
| 5. Authenticate  | Check session/onboarding/reauth | Route to auth flow         |
| 6. Transcribe    | If voice: download, Whisper STT | Ask to resend or type      |
| 7. Process       | Send to Gemini with tools       | Retry once, then error     |
| 8. Execute tools | Run function calls              | Return error to Gemini     |
| 9. Route         | Send to kitchen members by role | Retry with backoff         |
| 10. Log          | Write event_log, update Redis   | Warn but do not block      |
